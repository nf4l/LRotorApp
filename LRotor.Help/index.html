Using LRotor

LRotor is designed to turn a rotor that uses a control box configured for serial port control.

If you use RUMlog, Tom DL2RUM has made it scriptable, and an Applescript to connect to it is included with this install. To use it, you must have RUMlog (V5.2.2 or higher). Run the script "RUMlog2LRotor_V3.2.app". It will start RUMLog and LRotor. When headings are shown in RUMlog, the "DCXCC:" and "Sta:" boxes in LRotor will be filled.

RUMLog uses grid locators to calculate headings, and if one is supplied for the station, headings are calculated for both the DXCC entity and the station. Sometimes when an operator is on a DxPedition, the grid locator supplied is that of his home station.

The first time LRotor starts, select the serial connection you want to use, and the rotor. Baud rate will be automatically set.When those items are set, LRotor will open the serial connection, and the LED will turn green. You may also set colors in Preferences>Misc. You can also set LRotor to stay on top of RUMlog itself or other windows.On subsequent startup, LRotor will remember the port, rotor, colors and where it was on your screen.

If you use LRotor as a standalone program turn your antenna by typing in the heading and clicking the "Turn" or "LP" button. Click the "LP" (Long Path) button to turn the antenna 180o from the heading in the box. The button title toggles to "SP" to go back to short path. It's set to "LP" when the "Turn" button is clicked.

You can put the script anywhere you like and put an icon on the dock. Running the script will start RUMlog and LRotor. If LRotor is not visible, click on it's icon on the dock.
